# README Generator Skill

현재 프로젝트의 README.md를 자동으로 생성합니다.

## 실행 순서

### 1단계: 구성 방식 선택

사용자에게 구성 방식을 선택하도록 요청합니다:

```
어떤 방식으로 README를 구성할까요?

[1] 기본 양식 — 프로젝트 개요, 기술 스택, 시작하기, 라이선스
[2] 커스텀 구성 — 섹션을 직접 선택합니다
```

### 2단계: 커스텀 구성 시 섹션 선택

커스텀 선택 시 다음 섹션 목록을 보여주고 포함할 섹션을 선택받습니다:

```
[ ] 프로젝트 개요 + 배지
[ ] 스크린샷 / 데모 GIF
[ ] 주요 기능
[ ] 기술 스택 표
[ ] 시작하기 (설치 + 실행)
[ ] 환경변수 목록 (.env 항목)
[ ] 프로젝트 폴더 구조
[ ] API 문서
[ ] 개발 가이드 (브랜치, 커밋, PR)
[ ] 배포 방법
[ ] 기여 방법
[ ] 라이선스
```

선택 후 확인을 받고, 원하면 수정할 수 있게 합니다.

### 3단계: 각 섹션 정보 수집 방식 선택

선택된 섹션마다 정보 수집 방식을 묻습니다:

```
[섹션명] 정보를 어떻게 채울까요?
[1] 직접 입력
[2] 건너뛰기
[3] AI 자동 생성 (코드베이스 분석)
```

### 4단계: Sub-agents 병렬 실행

"AI 자동 생성"이 하나라도 선택된 경우, 다음 3개의 sub-agent를 **병렬로** 실행합니다:

- **Sub-agent 1**: 프로젝트 폴더 구조 분석 (tree 형태로 정리)
- **Sub-agent 2**: package.json, 설정 파일에서 기술 스택 파악
- **Sub-agent 3**: 주요 소스 파일 분석으로 핵심 기능 파악

### 5단계: README.md 생성

수집된 모든 정보를 합쳐 현재 프로젝트 루트에 `README.md`를 생성합니다.

생성 완료 후 파일 경로를 알려줍니다.
