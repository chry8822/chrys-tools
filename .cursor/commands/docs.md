# Docs

커밋 전, 현재 변경사항을 분석해 업데이트가 필요한 문서만 선별하여 자동으로 업데이트한다.

## 사용방법

```
/docs
```
→ 현재 변경사항을 분석해 업데이트 대상 문서를 자동 판단하고 수정

---

## 진행 순서

### STEP 1. 변경사항 파악

터미널에서 아래 명령어를 실행해 현재 변경 내용을 파악한다.

```
git status
git diff HEAD
```

### STEP 2. 문서 업데이트 대상 판단

STEP 1 결과를 바탕으로 아래 기준에 따라 업데이트가 필요한 문서를 선별한다.

| 변경 종류 | 업데이트 대상 |
|---|---|
| 신규 기능 구현, 버그 수정, UI/UX 개선, 애니메이션, 레이아웃 변경 | 프로젝트 내 작업 추적 문서 (예: `UI_INTERACTION_IMPROVEMENTS.md`) |
| 새 라이브러리 설치, 환경 변수 추가, 프로젝트 구조 변경, 실행 방법 변경 | `README.md` |
| 단순 텍스트/스타일 수정, 오타 수정, 주석 변경 | 업데이트 불필요 → STEP 3 생략 |

업데이트 대상이 없으면 아래를 출력하고 종료한다.
```
문서 업데이트가 필요한 변경사항이 없습니다.
/commit 을 바로 실행하세요.
```

### STEP 3. 대상 문서 확인 및 업데이트

업데이트 대상 문서가 프로젝트에 실제로 존재하는지 확인한다.
존재하는 문서만 업데이트하고, 없는 문서는 건너뛴다.

**작업 추적 문서 업데이트 규칙:**
- 완료된 항목은 `✅`, 스킵된 항목은 `⏭`으로 표시
- 이미 기록된 항목은 중복 추가하지 않고 상태만 갱신
- 신규 항목은 해당 섹션 하단에 추가
- 날짜 필드가 있으면 오늘 날짜로 갱신

**README 업데이트 규칙:**
- 설치/실행 방법, 환경 변수, 의존성 변경 내용만 반영
- 기존 구조를 유지하면서 변경된 부분만 수정

### STEP 4. 결과 보고

업데이트 완료 후 아래 형식으로 결과를 출력한다.

```
──────────────────────────────────
📄 문서 업데이트 완료

수정된 파일:
- UI_INTERACTION_IMPROVEMENTS.md  (변경 내용 요약)
- README.md  (변경 내용 요약)

이제 /commit 을 실행하세요.
──────────────────────────────────
```

---

## 규칙

- 변경사항과 무관한 문서는 절대 수정하지 않는다
- 존재하지 않는 문서는 새로 생성하지 않는다
- 이미 기록된 내용을 중복으로 추가하지 않는다
- 문서 업데이트만 하고 커밋은 하지 않는다 (커밋은 `/commit`으로 별도 진행)
