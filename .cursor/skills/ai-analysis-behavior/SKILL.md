# AI 분석 행동 규칙

## RULE 1: 답변 전 반드시 파일을 읽어라

- 코드 질문 시 반드시 Read 도구로 관련 파일을 직접 읽은 후 답변
- 기억이나 추측으로 답변 절대 금지
- 사용자가 코드 일부를 첨부했더라도 연관 파일(import, props, 호출 스택)까지 읽어 전체 맥락 파악
- 읽은 파일과 라인 번호를 근거로만 결론 도출

## RULE 2: 맥락 전체를 분석하라

분석 순서:
1. 문제 발생 파일 읽기
2. 의존 파일(import, 훅/서비스/컴포넌트) 읽기
3. 위에서 아래로 실행 흐름 추적
4. 흐름 추적 결과로 원인 도출

## RULE 3: 코드 시뮬레이션을 실행하라

원인을 단정하기 전에 반드시 아래 형식으로 실행 흐름을 추적한다:

```
[입력] 사용자 액션 또는 코드 진입점
  → [실행 1] 어떤 코드가 실행되는가 (파일명:라인)
  → [실행 2] 그 결과로 무엇이 호출되는가 (파일명:라인)
  → [결과] 최종적으로 어떤 상태/화면이 되는가
```

시뮬레이션 없이 "이게 원인입니다" 단정 금지.

## RULE 4: 보고서 형태로 답변하라

```
[분석 대상] 읽은 파일 목록 및 라인 범위
[흐름 파악] 연관 로직 의존 관계
[시뮬레이션] 코드 실행 흐름 추적 결과
[원인] 근거 코드 파일명:라인 명시
[해결 방법] 시뮬레이션 기반 해결책
[검증] 해결 방법 적용 후 재시뮬레이션
```

- 전문 용어 사용 시 한 줄 부연 설명 추가
- 불확실한 경우 "확인이 더 필요합니다" 먼저 명시 후 추가 파일 요청

## RULE 11: UI 버그 시 props 소스를 먼저 추적하라

1. 컴포넌트가 받는 props 목록 확인
2. 각 prop이 어디서 오는지 추적 (store / context / 부모)
3. 버그 발생 액션 시점의 store 상태 변화 확인
4. 변경된 store → 컴포넌트 props 전달 경로 확인
5. props 값 기준으로 렌더 결과 시뮬레이션

store 확인 없이 컴포넌트 코드를 수정하는 것은 금지.
